# Simple-Medium Checklist

Use for bounded low-risk tasks. Every item must be marked `done`, `not_applicable: reason`, or `blocked: reason`.

## Checklist

- [ ] Refresh FPF spec and protocol repository; record commits and warnings.
- [ ] Classify the normalized task as `simple-medium`; state the scope in one sentence.
- [ ] Identify the active system(s), role(s), method/work distinction, and the expected output. Use FPF patterns A.1 Holonic Foundation, A.1.1 Bounded Context, A.2 Role Taxonomy, A.2.1 U.RoleAssignment, A 2.2 U.Capability, A.2.4 U.EvidenceRole, A.3.1 U.Method, A.3.3 U.Dynamics, A.4 Temporal Duality, A.2.6 Unified Scope Mechanism, all cluster A.6 patterns (especially A.6.P U.RelationalPrecisionRestorationSuite), A.7 Strict Distinction, A.10 Evidence Graph, A.11 Ontological Parsimony, A.15 Role-Method-Work Alignment, C.7 CHR‑CAL – Characterisation Kit, C.27 Temporal Claim Adequacy, all cluster E.17 patterns and other relevant FPF patterns to structure your answer.
- [ ] Check whether external evidence is needed; if yes, use admissible sources or escalate to `complex`.
- [ ] Answer in the user's domain language without unnecessary FPF jargon.
- [ ] Separate facts, assumptions, hypotheses, and recommendations.
- [ ] Verify the answer for internal consistency, temporal adequacy, and no invented facts.
- [ ] Add a concise engineering basis when the answer is substantive.
- [ ] Mark completion: `Simple-medium checklist complete`.
