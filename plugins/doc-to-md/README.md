# Doc To Md Plugin

This plugin distributes the public `doc-to-md` Codex skill as a source artifact.
The bundled skill includes reusable workflow profiles for ordinary local
documents and textbook-like PDF audit/OCR work. Personal defaults belong in a
private local policy file, not in this public plugin.

## What Is Included

- `skills/doc-to-md/README.md`
- `skills/doc-to-md/SKILL.md`
- `skills/doc-to-md/agents/`
- `skills/doc-to-md/scripts/`
- `skills/doc-to-md/references/`
- `skills/doc-to-md/requirements-*.txt`
- `skills/doc-to-md/requirements-*.hashes.txt`
- `skills/doc-to-md/THIRD_PARTY_NOTICES.md`

## What Is Not Included

This plugin intentionally does not include local runtime state:

- Python venvs under `${CODEX_HOME:-$HOME/.codex}/tools`;
- wrapper symlinks or installed commands from `~/.local/bin`;
- Tesseract, Ghostscript, Homebrew packages, or other machine-specific binaries;
- private local policy files, private overlays, or user-specific skill forks;
- private source documents, local fixtures, OCR PDFs, audit bundles, generated Markdown, extracted assets, cache, logs, or temp files.

Those are local operational infrastructure or generated outputs. They must be installed, rebuilt, or generated by the user after plugin installation.

## Runtime Contract

The bundled skill source is not a ready-to-run conversion environment by itself.
After installing the plugin, run the bundled installer from the installed skill
directory:

```bash
bash "${CODEX_HOME:-$HOME/.codex}/skills/doc-to-md/scripts/install.sh"
```

For Claude Code or another non-`.codex` skill location, run the same installer
from that installed source path. The installer writes command shims that record
`DOC_TO_MD_SKILL_DIR` and delegate to the bundled source wrappers:

```bash
export DOC_TO_MD_SKILL_DIR="$HOME/.claude/skills/doc-to-md"
export DOC_TO_MD_BIN_DIR="${DOC_TO_MD_BIN_DIR:-$HOME/.local/bin}"
export DOC_TO_MD_TOOLS_DIR="${DOC_TO_MD_TOOLS_DIR:-$HOME/.codex/tools}"
bash "$DOC_TO_MD_SKILL_DIR/scripts/install.sh"
```

Optional workflows are installed explicitly:

```bash
bash "${CODEX_HOME:-$HOME/.codex}/skills/doc-to-md/scripts/install.sh" --book
bash "${CODEX_HOME:-$HOME/.codex}/skills/doc-to-md/scripts/install.sh" --ocr
bash "${CODEX_HOME:-$HOME/.codex}/skills/doc-to-md/scripts/install.sh" --all
```

For stricter public release on the maintained macOS arm64 / Python 3.13 profile:

```bash
bash "${CODEX_HOME:-$HOME/.codex}/skills/doc-to-md/scripts/install.sh" --hash-locked
```

For stricter public release on the maintained Intel macOS / Python 3.12 core
and book profile:

```bash
PYTHON=python3.12 DOC_TO_MD_HASH_PROFILE=macos-intel-py312 bash "${CODEX_HOME:-$HOME/.codex}/skills/doc-to-md/scripts/install.sh" --hash-locked
PYTHON=python3.12 DOC_TO_MD_HASH_PROFILE=macos-intel-py312 bash "${CODEX_HOME:-$HOME/.codex}/skills/doc-to-md/scripts/install.sh" --book --hash-locked
```

External OCR tools such as Tesseract remain separate local dependencies and are
checked by `mdown-ocrpdf --doctor`.

Python minor versions are profile-specific. A profile such as
`macos-arm64-py313` does not imply support for `macos-arm64-py312` or
`macos-arm64-py314`; unlisted profiles are candidate/unverified until the
bundled `skills/doc-to-md/references/python-profiles.md` lists them as
supported.

The public workflow profiles are reusable guidance, not personal defaults. For
example, `eng+rus` is documented for English/Russian OCR when the relevant
language packs are installed; it is not a universal OCR default.

User-facing diagnostics use the public diagnostic shape documented in the
bundled `skills/doc-to-md/references/diagnostics.md` file.

## Support Boundary

Codex on macOS arm64 is supported for core, book, and OCR. Codex on Intel macOS
is supported for core and book on Python 3.12; Intel OCR hash-locked support is
not published. WSL is a candidate. Claude Code on macOS is experimental unless
installer-generated shims record `DOC_TO_MD_SKILL_DIR`, `DOC_TO_MD_BIN_DIR`,
and `DOC_TO_MD_TOOLS_DIR`.
Native Windows PowerShell/CMD is unsupported.

Read `skills/doc-to-md/references/support-matrix.md`,
`skills/doc-to-md/references/python-profiles.md`, and
`skills/doc-to-md/references/scenario-boundaries.md` before installing outside
the maintained Codex/macOS path or promising public support for a scenario.

## Validation

From the repository root:

```bash
scripts/validate-plugins.sh
scripts/validate-skills.sh
scripts/validate-doc-to-md-release.sh
```

After installing runtime dependencies locally:

```bash
mdown-doctor
mdown-book --doctor
mdown-ocrpdf --doctor
```
